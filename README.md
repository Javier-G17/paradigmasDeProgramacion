## Descripción
Este proyecto consiste en el desarrollo de un sistema básico de gestión de mascotas utilizando dos paradigmas de programación en Python.

## Programa 1: Programacion Tradicional
Este programa fue desarrollado utilizando variables, funciones y entradas de datos por teclado. Permite registrar los datos de una mascota (nombre, especie y edad) y mostralos de forma organizada sin utilizar clases ni objetos. 

## Programa 2: Programación Orientada a Objetos 
Este programa fue desarrollado utilizando un aclase llamada Mascota. La clase contiene los atributos nombre, especie y edad, además de los métodos mostar_informacion() y hacer_sonido().
En el programa principal se crean objetos de la clase y se ejecutan sus métodos para mostar la información de cada mascota.

## Reflexión
La principal diferencia encontrada entre la Programación Tradicional y la Programación Orientada a Objetos es la forma en que se organiza el código. 
En la programación tradicional, los datos y las funciones se manejan por separado, lo que resulta adecuado para programas pequeños y sencillos. 
En cambio, la programación orientada  a objetos permite agrupar datos y comportamientos dentro de una misma clase, facilitando la organización, reutilización y mantenimiento.
Durante el desarrollo de esta actividad se pudo observar que la programación orientada a objetos ofrece una estructura más escalable y ordenada, especialmente cuando se trabaja con múltiles entidades que comparten características y comportamientos similares. Sin embargo para problemas simples, la programación tradicional puede ser más rapida de implementar y comprender
